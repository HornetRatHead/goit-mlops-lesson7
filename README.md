ArgoCD Deployment for MLOps Infrastructure
Цей проект призначений для автоматизованого розгортання ArgoCD у кластері Amazon EKS за допомогою Terraform та Helm. ArgoCD виступає як GitOps-інструмент для подальшого деплою ML-сервісів (MLflow, Tracking Server тощо).

!!! Важлива примітка: Попередні вимоги
1) Кластер Kubernetes (EKS) не створюється цим кодом. Інфраструктура кластера повинна бути піднята окремо. Якщо у вас ще немає     готового кластера, ви можете скористатися базовим шаблоном інфраструктури за посиланням: https://github.com/HornetRatHead/goit-mlops-lesson5-6

!!!Увага:
Якщо кластер підіймається на t3.small потрібно щоякнайменше 3 ноди, інакше їх не вистачить

Архітектура та зв'язок проектів
Цей проект використовує механізм Terraform Remote State. Це дозволяє ArgoCD автоматично отримувати необхідні дані (endpoint кластера, CA сертифікат) безпосередньо з S3-бакета, куди зберіг свій стан проект EKS.

2) Налаштування "Містка" (Remote State)
У файлі data.tf необхідно вказати шлях до стейту вашого кластера:

data "terraform_remote_state" "eks" {
  backend = "s3"
  config = {
    bucket = "ваш-terraform-state-bucket" # Назва вашого S3 бакета
    key    = "eks/terraform.tfstate"      # Шлях до стейту з проекту lesson5-6
    region = "eu-central-1"               # Регіон, де лежить бакет
  }
}


!!!Увага:
Треба перевірити actual location S3 бакета. Переконатися, що в backend.tf вказано правильний регіон саме для бакета

Порядок розгортання
Підготовка кластера: Переконайтеся, що кластер EKS створено та він має статус ACTIVE.

2) Налаштування доступу:
aws eks update-kubeconfig --region eu-central-1 --name <назва-вашого-кластера>

---------------------------------------------------------------------------------------------------------
  
3)Ініціалізація Terraform:
  
Оскільки ми використовуємо S3 як бекенд, спочатку запустіть:
terraform init -reconfigure

Перевірка плану:
terraform plan

Застосування конфігурації:
terraform apply

FAQ

Виправлення типових помилок:

Помилка 301 (Moved Permanently)
Якщо ви отримуєте помилку 301, це означає невідповідність регіону S3-бакета в конфігурації backend.tf або data.tf. Перевірте фізичне розташування бакета командою:
aws s3api get-bucket-location --bucket <назва>

Pending Status (Insufficient Resources)
Для стабільної роботи ArgoCD на маленьких інстансах (наприклад, t3.small), переконайтеся, що у вашому кластері щонайменше 3-4 ноди. В іншому випадку поди можуть зависнути у стані Pending через ліміти ENI (IP-адрес).

---------------------------------------------------------------------------------------------------------

4)Доступ до ArgoCD після встановлення
Після успішного apply, виконайте наступні кроки:

Отримайте пароль адміністратора:
kubectl -n infra-tools get secret argocd-initial-admin-secret -o jsonpath="{.data.password}" | base64 -d

Прокиньте порт для доступу до UI:
kubectl port-forward svc/argocd-server -n infra-tools 8080:443

Відкрийте браузер за адресою https://localhost:8080.

----------------------------------------------------------------------------------------------------------

5) Видалення інфраструктури

Для повного видалення ArgoCD:
terraform destroy


!!! Примітка: Це видалить лише ArgoCD та його компоненти. Сам кластер EKS залишиться активним.
